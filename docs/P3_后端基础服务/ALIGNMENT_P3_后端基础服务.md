# ALIGNMENT_P3_后端基础服务

> 本文件用于 P3 阶段**需求对齐**，严格基于 PRD §3.5.1 基础服务层。

## 一、PRD 原文回溯（§3.5.1）

1. **统一认证授权服务**：多端统一用户认证、OAuth2.0 权限体系、token 管理、接口鉴权
2. **用户中心服务**：用户/商户/骑手账号管理、实名认证、资质信息管理、地址管理
3. **消息推送服务**：APP 推送、微信小程序订阅消息、短信通知、站内信统一分发
4. **文件存储服务**：图片、视频、凭证等文件上传、存储、管理，对接对象存储服务
5. **地图与定位服务**：对接高德/百度地图开放平台，提供定位、地址解析、路径规划、配送范围校验、骑手实时定位上报与存储

## 二、相关的非功能性要求
- 接口平均 ≤ 200ms（PRD §4.1.1）
- 骑手定位上报 ≥ 10s/次（PRD §4.1.4）
- 敏感信息加密存储、全链路 HTTPS（PRD §4.3.1）
- 接口鉴权、防重放、防篡改、频率限制（PRD §4.3.2）
- 验证码防护、多端登录风控（PRD §4.3.4）

## 三、P3 任务范围

### 3.1 in-scope
| 服务 | 核心交付 |
|---|---|
| Auth | JWT + Refresh Token + 多端 Guard + 微信小程序 code2session + 商户/骑手账密 + 管理员账密 + 短信验证码 + 登录风控 + 权限守卫 |
| User | CRUD 用户/商户/骑手/管理员；实名认证 & 资质提交审核；地址 CRUD；黑名单 |
| Message | 抽象 MessageChannel；微信订阅消息、极光/个推 APP 推送、阿里云/腾讯云短信、站内信（DB + Redis + 广播）；模板管理；重试 |
| File | MinIO/OSS 适配层；后端签名直传（STS）；图片水印；病毒/MIME 校验；`file_meta` 入库；CDN URL 返回 |
| Map | 高德地图封装（地理编码、逆地理、距离矩阵、路径规划）；配送范围校验（点在多边形内）；骑手位置上报（Redis GEO + TimescaleDB） |

### 3.2 out-of-scope
- 具体业务流程（订单、支付、派单、营销）→ P4
- 前端对接 → P5~P8
- 压测、K8s 部署 → P9

## 四、关键约束对齐
| 约束 | 响应 |
|---|---|
| 多端统一认证 | 单 JWT 结构、`userType` 字段区分，Guard 装饰器级别区分 |
| Token 管理 | Access 2h + Refresh 30d；Refresh 存 Redis，可吊销 |
| 鉴权粒度 | 菜单级 + 按钮级 + 接口级（permission.code） |
| 敏感存储 | AES-GCM（P2 已设计） |
| 接口防刷 | `@Throttle` + IP/userId 限流；签名头 X-Sign（管理后台） |
| 防重放 | 时间戳 + Nonce（Redis SETNX，TTL 60s） |
| 文件安全 | 签名 URL、白名单扩展名、≤ 20MB |
| 定位上报 | 10s/次，Redis GEO 热；批处理写 TimescaleDB |
