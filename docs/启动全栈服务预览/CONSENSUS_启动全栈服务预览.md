# 共识文档 (CONSENSUS) - 启动全栈服务预览

## 1. 明确的需求描述
- **目标**：一键拉起整个平台的所有前端开发服务，配合已经在运行的后端，实现全栈预览。
- **服务清单**：
  1. `后端` (已在运行，端口 :3000)
  2. `管理后台` (Web 版 Vite，预计端口 :5173 左右)
  3. `用户端` (将其编译为 H5 以便于浏览器直接预览)
  4. `商户端` (编译为 H5 以便于预览)
  5. `骑手端` (编译为 H5 以便于预览)

## 2. 验收标准
- [ ] 管理后台能够无报错启动开发服务器，并给出 Local 访问地址。
- [ ] 用户端、商户端、骑手端能够通过 `pnpm dev:h5` 成功启动，并分别给出对应的预览端口。
- [ ] 各个进程在后台持续稳定运行，且不会互相冲突（端口独立）。

## 3. 边界与技术约束
- 为了方便在没有微信开发者工具和手机模拟器的情况下进行调试，三端（用户、商户、骑手）均采用 `dev:h5` 命令启动。
- 因为 4 个服务同时启动日志输出极大，所有启动命令均将在后台异步运行，您可以通过进程状态或直接访问对应的 localhost 端口进行访问。
