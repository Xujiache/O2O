## 验证方法（T6.49）

### 1. 触发覆盖

按钮逐一点击，确保每个 eventId 至少触发 1 次：
- 登录 → 入驻 → 审核 → 工作台
- 切店铺 → 编辑信息 → 配送范围 → 评价回复
- 接单 → 出餐 → 打印 → 退款审核 → 异常上报
- 新建商品 → 编辑 → 上下架 → 限时折扣
- 提现 → 导出账单 → 申请发票
- 创建店铺券 / 活动 / 子账号

### 2. 离线持久化

- 切飞行模式 → 触发若干事件
- kill 进程 → 重启
- **预期**：BUFFER 从 storage 恢复，恢复网络后自动上报

### 3. 上报验证

后端控制台 / Sentry breadcrumbs（如启用）应能看到对应事件。

---

## 验收标准

- ✅ 所有 45 个事件至少触发 1 次
- ✅ 离线触发 + 重启 + 联网后成功上报（无丢失）
- ✅ 单 batch 上限 20 / 10s flush
