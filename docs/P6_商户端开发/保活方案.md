# O2O 商户端 · 保活方案文档

> **目的**：商户在 APP 后台 / 锁屏 / 30min 静默时仍能稳定收到新订单推送 / WebSocket 消息  
> **覆盖**：iOS / Android（小程序版受限于平台政策不在范围）  
> **对应**：T6.41 Android Foreground Service / T6.42 iOS 静音音频保活

---

## 一、整体策略：四道防线

| 通道 | 触发场景 | 实现 |
|---|---|---|
| ① WebSocket | APP 在前台 / 后台 + 系统未挂起 | `utils/ws.ts` 心跳 30s + 指数退避重连 5 次 |
| ② 极光推送 | APP 被系统挂起 / APP 杀进程 | `utils/jpush.ts` token 注册 + `onJPushNotificationClick` 跳详情 |
| ③ Android 前台服务 | Android 锁屏 + 长时间静默 | `utils/foreground-service.ts` + nativePlugin `Mchnt-ForegroundService` |
| ④ iOS 静音音频 | iOS 后台 + 锁屏 | `utils/ringtone.ts` `startSilentAudio()` + `Info.plist UIBackgroundModes: audio` |

## 二、Android 前台服务（Foreground Service）

### 2.1 必要前提


### 2.2 nativePlugin 编写（5+ Android）

**插件名**：`Mchnt-ForegroundService`  
**目录**（HBuilderX 编译时）：`nativePlugins/Mchnt-ForegroundService/`

#### 2.2.2 `FgsModule.java`（核心）

```java
package com.o2omchnt.fgs;

import android.app.NotificationChannel;
import android.app.NotificationManager;
import android.content.Context;
import android.content.Intent;
import android.os.Build;
import com.alibaba.fastjson.JSONObject;
import io.dcloud.feature.uniapp.annotation.UniJSMethod;
import io.dcloud.feature.uniapp.bridge.UniJSCallback;
import io.dcloud.feature.uniapp.common.UniModule;

public class FgsModule extends UniModule {

  @UniJSMethod(uiThread = true)
  public void start(JSONObject opt, UniJSCallback callback) {
    Context ctx = mUniSDKInstance.getContext();
    if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.O) {
      NotificationChannel channel = new NotificationChannel(
        "o2o_mchnt_fgs",
        "接单服务",
        NotificationManager.IMPORTANCE_HIGH
      );
      channel.setDescription("商户端接单常驻通知");
      ((NotificationManager) ctx.getSystemService(Context.NOTIFICATION_SERVICE))
        .createNotificationChannel(channel);
    }
    Intent intent = new Intent(ctx, FgsService.class);
    intent.putExtra("title", opt.getString("title"));
    intent.putExtra("content", opt.getString("content"));
    if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.O) {
      ctx.startForegroundService(intent);
    } else {
      ctx.startService(intent);
    }
    if (callback != null) callback.invoke("ok");
  }

  @UniJSMethod(uiThread = true)
  public void stop(UniJSCallback callback) {
    Context ctx = mUniSDKInstance.getContext();
    ctx.stopService(new Intent(ctx, FgsService.class));
    if (callback != null) callback.invoke("ok");
  }
}
```

#### 2.2.3 `FgsService.java`

```java
package com.o2omchnt.fgs;

import android.app.Notification;
import android.app.PendingIntent;
import android.app.Service;
import android.content.Intent;
import android.os.IBinder;
import androidx.core.app.NotificationCompat;

public class FgsService extends Service {
  @Override
  public int onStartCommand(Intent intent, int flags, int startId) {
    String title = intent.getStringExtra("title");
    String content = intent.getStringExtra("content");
    Notification notification = new NotificationCompat.Builder(this, "o2o_mchnt_fgs")
      .setSmallIcon(android.R.drawable.ic_notification_overlay)
      .setContentTitle(title)
      .setContentText(content)
      .setOngoing(true)
      .setPriority(NotificationCompat.PRIORITY_HIGH)
      .build();
    startForeground(1001, notification);
    return START_STICKY;
  }

  @Override
  public IBinder onBind(Intent intent) {
    return null;
  }
}
```

### 2.3 验收

1. APK 打包 release 版（HBuilderX → 云打包）
2. 安装到 Android 8 / 10 / 13 真机
3. 登录后通知栏出现「O2O 商户端 · 接单服务 / 正在接收新订单，请保持常驻」

6. **预期**：30min 内 100% 收到，弹窗 + 铃声 + TTS 同步触发

## 三、iOS 静音音频保活

### 3.1 原理

iOS 系统对后台 APP 严苛挂起，只有 `UIBackgroundModes` 包含 `audio` / `voip` 的应用才能持续后台运行。我们用 **0 分贝静音音频循环播放** 欺骗系统不挂起 APP，使 WebSocket 心跳得以维持。

### 3.2 配置

`manifest.json` `app-plus.distribute.ios`：

```json
"UIBackgroundModes": ["audio", "fetch", "remote-notification"]
```

### 3.3 静音 wav 资源

路径：`商户端/src/static/audio/silent.wav`  
要求：单声道 / 8kHz / 16bit / **音量 0** / 时长 ≥ 1 秒（loop=true 即可）

> 由于资源是二进制，本仓库未直接入库该文件。**P9 阶段** 由音频工程师补录或从公开 silent wav 替换。

### 3.4 实现

`商户端/src/utils/ringtone.ts` 已暴露：

```ts
startSilentAudio()  // 仅 iOS 生效；Android / 小程序自动跳过
stopSilentAudio()
```

### 3.5 启动时机

`App.vue` `onLaunch()` 中：

```ts
if (authStore.isLogin) {
  startSilentAudio()  // iOS only
}
```

退出登录 / `onHide` 不主动停（让系统进入后台后保持）。

### 3.6 验收（≥ 80% 成功率，iOS 沙盒不能 100%）

1. iOS 13 / 15 / 17 真机各 1 部
2. 登录后切到后台 / 锁屏
3. 30min 后服务端推送新订单
4. **预期**：≥ 80% 成功率，弹窗 + 铃声同步

## 四、极光推送（兜底通道）

### 4.1 适用场景

- APP 被系统杀进程
- iOS 静音音频保活失效
- WebSocket 长时间断连

### 4.2 配置

`商户端/src/utils/jpush.ts` 已实现：
- `registerJPush()` 登录后注册 RegistrationId 至后端
- `onJPushNotificationClick()` 监听通知栏点击 → 跳订单详情

后端推送约定（见 P4 文档）：

```json
{
  "registrationId": "xxx",
  "alert": "您有新订单",
  "extras": {
    "type": "merchant:order:new",
    "orderNo": "TO20260418001",
    "shopId": "sp-1001"
  }
}
```

### 4.3 验收

1. 杀进程 APP（iOS 上滑 / Android 多任务关闭）
2. 服务端触发推送
3. **预期**：3 端均能弹横幅；点击通知 → 跳订单详情页

## 五、用户操作指引（必须告知商户）

### 5.1 Android

1. **首次安装**：弹出「允许通知 / 允许悬浮窗 / 允许后台运行」全部点击「允许」
2. **关闭电池优化**：设置 → 电池 → APP 电池优化 → O2O 商户端 → 「不优化」
3. **加入白名单**（部分国产 ROM）：
   - 华为 EMUI：手机管家 → 应用启动管理 → O2O → 手动管理 → 全部开
   - 小米 MIUI：设置 → 应用设置 → 应用管理 → O2O → 自启动 ✓ + 后台弹出 ✓
   - OPPO ColorOS：电池 → 应用耗电管理 → O2O → 允许后台运行
   - vivo：i 管家 → 后台高耗电 → O2O → 允许

### 5.2 iOS

1. 设置 → 通知 → O2O 商户端 → **打开通知 + 横幅 + 锁屏 + 声音**
2. 设置 → O2O → **后台 APP 刷新 = 开**
3. 不要在 iOS 多任务管理中**手动上滑**关闭 APP

## 七、已知限制

| 限制 | 影响 | 缓解 |
|---|---|---|
| iOS 静音保活非 100% | iOS 13+ 偶发挂起 | 极光推送兜底 |
| 小程序无 TTS / 铃声循环 | 商户在小程序版收单需主动看消息 | 提示商户优先用 APP |
| Android 14 通知权限收紧 | targetSdk=34 需运行时申请 POST_NOTIFICATIONS | manifest 已声明，APP 启动时调 `uni.authorize` |
| 后端 WS 心跳超时 60s | 网络抖动可能丢消息 | 客户端心跳 30s + 极光兜底 |
