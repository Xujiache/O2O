# ACCEPTANCE_P4_后端业务服务

## 一、功能验收（按模块）

### Shop + Product
| 编号 | 场景 | 标准 |
|---|---|---|
| V4.1 | 店铺 CRUD | 审核通过后可见；非营业时自动打烊；临时歇业生效 |
| V4.2 | 配送范围 | polygon 内可下单；外拒绝 `10101` |
| V4.3 | 店铺列表 | 距离/销量/评分/价格排序正确；分页 ≤ 500ms |
| V4.4 | 商品 CRUD + 上下架 | 上架后小程序可见；下架立即不可下 |
| V4.5 | 库存 | 并发下单 100 笔，SKU stock=50 → 50 成功 50 失败 |
| V4.6 | 套餐 | 套餐内 SKU 各自扣减 |

### Marketing
| V4.7 | 优惠券领取 | 达到 per_user_limit 拒领 |
| V4.8 | 下单券计算 | 满减/折扣/立减正确；互斥生效；最优推荐 |
| V4.9 | 新客礼 | 注册事件触发发券 |
| V4.10 | 拼单 | 达人数合并订单成功 |
| V4.11 | 红包 | 平均/随机分发正确、不超发 |

### Order 核心
| V4.12 | 外卖下单→支付 | 15min 未支付自动取消，库存回滚 |
| V4.13 | 商户接单→出餐 | 状态机正确；状态日志完整 |
| V4.14 | 骑手取件→送达 | 取件码核验通过；送达凭证上传 |
| V4.15 | 用户确认收货 | 状态→50；触发分账 T+1 |
| V4.16 | 商户拒单 | 状态→60；触发退款 |
| V4.17 | 跑腿帮送 | 价格预估 = 实际下单；取件码生效 |
| V4.18 | 跑腿帮买 | buy_list + 预算正确记录；到达后结算 |
| V4.19 | 跑腿帮取 | 逆向流程跑通 |
| V4.20 | 跑腿帮排队 | 服务时长计费正确 |
| V4.21 | 异常上报 | 凭证入库；进入工单 |

### Payment
| V4.22 | 微信 JSAPI | 返回预支付参数；回调触发 OrderPaid |
| V4.23 | 支付宝 | 同上 |
| V4.24 | 余额支付 | 余额不足拒绝；充足扣减 |
| V4.25 | 退款 | 同步/异步回调成功；账户流水正确 |
| V4.26 | 对账 | 当日对账单入库；差异订单打标 |

### Dispatch
| V4.27 | 系统派单 | 15s 超时自动 next；候选耗尽进抢单池 |
| V4.28 | 抢单 | 原子抢占，不会一单被抢两次 |
| V4.29 | 顺路单 | 顺路候选排序更高；骑手可批量接 |
| V4.30 | 转单 | 审核通过后原骑手释放；新骑手接管 |

### Finance
| V4.31 | 分账 T+1 | 定时任务准时触发；商户/骑手/平台三方金额正确 |
| V4.32 | 提现 | 申请→审核→打款 全链路；失败退回余额 |
| V4.33 | 发票 | 申请→开票→邮件送达 |
| V4.34 | 对账报表 | 生成 + 导出 Excel |

### Review
| V4.35 | 评价 | 一单一评；24h 内可改；含多媒体 |
| V4.36 | 申诉 | 商户/骑手可申诉；平台审核改判 |
| V4.37 | 投诉 & 工单 | 分派到客服；闭环处理 |
| V4.38 | 仲裁 | 结果自动触发退款/补偿；账户流水正确 |

### 闭环
| V4.39 | 外卖端到端 | 9 节点全流程闭环，无需人工 |
| V4.40 | 跑腿端到端 | 8 节点全流程闭环 |

## 二、非功能验收

| 项 | 标准 |
|---|---|
| 下单接口 P95 | ≤ 500ms |
| 订单列表 P95 | ≤ 800ms |
| 订单状态推送 | ≤ 3s（事件产生 → 前端 WS） |
| 支付回调处理 | ≤ 200ms |
| 派单决策 | ≤ 500ms |
| 单元测试覆盖 | ≥ 70% |
| 集成测试 | 2 大闭环用例通过 |
| Swagger | 接口/错误码/示例齐全 |

## 三、数据一致性
- 状态、支付、配送在 user/merchant/rider/admin 4 端数据一致（PRD §6.1.3）
- 订单状态日志每次变迁都有记录
- 账户流水每次变化都有流水
- 分账明细可对账
