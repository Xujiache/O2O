# 共识文档 (CONSENSUS) - 环境准备与后端启动

## 1. 明确的需求描述
- **环境重置**：彻底卸载本地现有的所有 MySQL 相关组件（包括 MySQL Server 8.0, Installer, Workbench 等），并删除相关数据目录（直接清空，无需备份）。
- **环境重装**：重新安装 MySQL Server 8.0，配置为开机自启的 Windows 服务（服务名 `MySQL80` 或默认），并将 `root` 账户密码设置为 `Mm8822775`。
- **验证依赖**：确认本地 Redis 服务已正常运行。
- **启动验证**：在根目录下执行 `pnpm install` 后，进入 `后端` 目录启动 NestJS 服务（`pnpm run start:dev`），确保其能够正常连接至新建的 MySQL 与现有的 Redis 服务，并且能够正常访问 Swagger 接口文档或健康检查接口。

## 2. 验收标准
- [ ] 系统中不再存在旧的 MySQL 安装文件及数据文件残留。
- [ ] 全新的 MySQL 8.0 服务在本地 3306 端口正常运行。
- [ ] 可以使用 `root` 账户及密码 `Mm8822775` 成功登录 MySQL。
- [ ] 本地后端服务能够无报错启动，成功连接至 `localhost:3306` (MySQL) 和 `localhost:6379` (Redis)。
- [ ] 可通过 `http://localhost:3000/docs` 或 `http://localhost:3000/health` 正常访问。

## 3. 边界与技术约束
- 卸载过程中采取强制静默卸载（使用 MSI 产品代码），避免交互式弹窗。
- 考虑到 MSI 安装和配置较为耗时，为了实现“彻底重装”，将通过下载官方解压版（ZIP）或者调用 `msiexec` 进行静默安装，以确保密码的确定性配置。我们将采用 ZIP 解压版安装方式作为重装方案（由于其绿色环保，易于配置且可精确控制密码和数据目录）。
- 后端服务的环境变量 `.env.development` 保持现有配置，不做变更（其中定义了 DB 密码和 Redis 配置）。
