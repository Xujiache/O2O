# ALIGNMENT_P9_集成测试部署

> 本文件用于 P9 阶段**需求对齐**，严格基于 PRD §4（非功能性需求）+ §6（验收标准）。

## 一、PRD 原文回溯

### 1.1 性能要求（§4.1）
1. 核心接口平均 ≤ 200ms，峰值 ≤ 500ms
2. 订单并发 ≥ 1000 TPS，高峰无卡顿、无数据丢失
3. 小程序首屏 ≤ 2s，页面切换 ≤ 300ms
4. 骑手定位 ≥ 10s/次；订单状态推送延迟 ≤ 3s
5. 百万级订单数据查询 ≤ 1s

### 1.2 兼容性（§4.2）
1. 微信小程序基础库 ≥ 2.0
2. Android 8.0+ / iOS 13.0+
3. Chrome、Edge、Firefox 最新 2 个版本
4. uni-app 多端一致性

### 1.3 安全（§4.3）
1. 敏感信息加密 + 全链路 HTTPS
2. 接口鉴权、防重放、防篡改、频率限制
3. 支付全链路加密、资金双人复核、异常风控拦截、对账完善
4. 多端账号风控、异常登录提醒、验证码防护、权限最小化
5. 合规：《个人信息保护法》《网络安全法》

### 1.4 可用性（§4.4）
1. 核心服务 ≥ 99.9%，年 downtime ≤ 8.76h
2. 故障自动切换，恢复 ≤ 30min，数据无丢失
3. 全量 + 增量备份，异地存储
4. 高峰期非核心功能降级

### 1.5 易用性（§4.5）
覆盖各端使用体验指标

### 1.6 验收（§6）
1. **§6.1 功能验收**：PRD 功能 100% 覆盖；外卖/跑腿闭环跑通；四端数据一致；异常处理完善
2. **§6.2 性能与兼容性验收**：满足 §4.1/§4.2；小程序通过合规检测；APP 可上架
3. **§6.3 安全与可用性验收**：第三方扫描无高危中危；72h 稳定运行；备份可恢复
4. **§6.4 交付物验收**：源代码 / 数据库脚本 / 部署脚本 / 编译包；完整文档；测试用例与报告

## 二、P9 任务范围

### 2.1 in-scope
| 类别 | 内容 |
|---|---|
| 测试 | 单元、集成、E2E、性能（JMeter/k6）、安全（OWASP/ZAP）、兼容（BrowserStack/真机） |
| 容器化 | Dockerfile（后端/管理后台）+ docker-compose（生产） |
| 编排 | Kubernetes Manifests（Deployment/Service/Ingress/ConfigMap/Secret/HPA）+ Helm Chart |
| CI/CD | Jenkins Pipeline（lint→test→build→scan→push→deploy→smoke） |
| 监控 | Prometheus 指标采集 + Grafana 面板 + AlertManager |
| 日志 | ELK（Elasticsearch + Logstash + Kibana）或 Loki + Grafana |
| 反向代理 | Nginx 配置、TLS 证书（Let's Encrypt / 企业证书） |
| 备份 | MySQL xtrabackup + Redis RDB/AOF + MinIO 异地复制 |
| 降级与容灾 | 开关中心、限流熔断（后端 Rate Limit + Redis） |
| APP 上架 | App Store + Android 主流市场（华为/小米/OPPO/VIVO） |
| 小程序上架 | 微信审核提交 |
| 文档 | 运维手册、故障 Runbook、应急响应、SLO/SLI |

### 2.2 out-of-scope
- 多区域多活（企业级，后续规划）
- BI/数仓建设（后续）
- 合规等保测评（独立项目）
