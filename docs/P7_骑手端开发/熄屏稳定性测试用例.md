# 熄屏 30 分钟定位稳定性测试用例（V7.22）

> 阶段：P7 / Sprint 7 / T7.48
> 编制：单 Agent V2.0
> 验收基准：ACCEPTANCE_P7 §一 V7.22

## 一、目标

骑手 APP 熄屏 30 分钟，**Foreground Service 持续上报 10s/次定位，单量不遗漏**。

## 二、前置环境

| 项 | 配置 |
|---|---|
| 设备 | 小米 13 / Redmi Note 12 / 华为 P50 / OPPO Find X5（4 款主流国产） |
| 系统 | Android 14（13/12 兼容验证） |
| 电量 | 80%+（避免电量降频干扰） |
| 网络 | 5G + WiFi 双通道 |
| 后端 | dev 环境 NestJS + MySQL + RabbitMQ |
| nativePlugin | Rider-LocationFgs **真编后挂载** |

## 三、测试步骤

### 3.1 准备

1. 安装含 Rider-LocationFgs nativePlugin 的 APK（HBuilderX 云打包）
2. 授予权限：定位（始终）+ 通知 + 后台运行 + 电池优化白名单
3. 关闭 MIUI / EMUI 等厂商"省电模式"
4. 关闭"自动管理后台应用"

### 3.2 用例步骤

```
1. 上班打卡 → 工作台显示在线
2. 启动 logcat 抓取（adb logcat | grep -E "LocationFgs|fgs.location")
3. 启动后端日志监听 POST /api/v1/map/rider/report
4. 锁屏（电源键短按）
5. 静置 30 分钟（不开屏、不操作）
6. 30 分钟后亮屏
7. 抓取以下日志：
   - 客户端 logcat: 检查每 10s 的 fgs.location 输出
   - 后端日志: 检查每 10s 一次 /map/rider/report POST
8. 解屏后立即"下班打卡"，触发 location.stop()
```

## 四、断言（V7.22）

| 断言 | 期望 | 实测 |
|---|---|---|
| Foreground Notification 持续可见 | 整个 30 分钟通知栏看得到"O2O 骑手端 · 接单服务" | ⚠️ 真机验 |
| 客户端心跳数（10s/次） | ~180 次 ± 10%（30 \* 60 / 10 = 180） | ⚠️ 真机验 |
| 后端接收次数 | ≥ 162 次（90% 成功率，允许网络抖动） | ⚠️ 真机验 |
| 单量不遗漏 | 期间收到的派单 100% 落 store + 弹 DispatchModal | ⚠️ 真机验 |
| 卡尔曼滤波生效 | 滤波后相邻点漂移 ≤ 15m（V7.19 已自动测） | ✅ 单测 PASS |
| WAKE_LOCK 持有 | adb shell dumpsys power 看到 PARTIAL_WAKE_LOCK 'O2ORider:LocationFgs' | ⚠️ 真机验 |
| 电量消耗 | 30min 减 ≤ 4%（线性外推 8h ≤ 30%） | ⚠️ 真机验 |

## 五、失败回退（如不满足）

| 失败现象 | 排查方向 | 修复 |
|---|---|---|
| 客户端心跳停止 | 厂商省电策略 | 引导用户加白名单 + 显示"建议开启"页 |
| Notification 消失 | foregroundServiceType 缺失 | 检查 manifest 已声明 `location` |
| 后端 0 接收 | 网络断 / token 过期 | 检查 offlineQueue 是否在收 |
| WAKE_LOCK 未持有 | acquireWakeLock 异常 | 检查 PowerManager 权限 |

## 六、自动化（P9 集成）

```bash
# adb 抓 30min 心跳 + 计数
adb logcat -c
adb shell input keyevent KEYCODE_POWER  # 锁屏
sleep 1800
adb shell input keyevent KEYCODE_POWER  # 亮屏
adb logcat -d | grep "fgs.location" | wc -l   # 应 ≥ 162
```

## 七、结论

代码层 / nativePlugin 已就位，**熄屏 30min 用例归 P9 真机执行**。
所有 Java 实现严格按 Android 14 FOREGROUND_SERVICE_TYPE_LOCATION 规范。
